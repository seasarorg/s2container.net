FAQ

  • Class specified in a dicon file can not be found and S2Container.NET throws ClassNotFoundRuntimeException error
  • Can not output S2Container.NET log
  • S2Unit.NET does not insert rows in order they are written in an Excel sheet
  • Dicon file can not be found

Class specified in a dicon file can not be found and S2Container.NET throws ClassNotFoundRuntimeException error

Followings are some possible reasons for this error:

  • Class name specified in the dicon file is misspelled.
  • Assembly is not specified in the [seasar] section in application configuration file (app.config).
    Document
    S2Container.NET reference - application configuration file

Can not output S2Container.NET log

Possible reason may be log4net is not setup or the log4net configuration file can not be read. Refer to the following page on setting up log4net.

Setup - log4net setup

S2Unit.NET does not insert rows in order they are written in an Excel sheet

This is a restriction of S2Unit.NET. Refer to the following page for details.

Restrictions

Dicon file can not be found

When dicon file is specified as an embedded resource ("namespace/Dicon file"), build action property in dicon file must be set to "Embedded resource" Refer to the following page for further details.

S2Container.NET - Creation of S2Container and Dicon configuration file
